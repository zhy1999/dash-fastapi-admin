# 登录认证接口文档

> 基础路径：`http://<后端IP>:38038`  
> 数据格式：JSON  
> 认证方式：Bearer Token（JWT）

---

## 一、用户登录

### 接口信息

| 项目 | 说明 |
|------|------|
| **URL** | `POST /login` |
| **Content-Type** | `application/x-www-form-urlencoded` |
| **是否需认证** | 否 |

> ⚠️ 注意：请求体是 `form-data` / `application/x-www-form-urlencoded`，不是 JSON。

---

### 请求参数（Form Data）

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `username` | string | ✅ | 用户名 |
| `password` | string | ✅ | 密码（明文传输，生产环境建议配合 HTTPS） |
| `code` | string | 条件 | 验证码（后端开启验证码时必填） |
| `uuid` | string | 条件 | 验证码会话ID（后端开启验证码时必填） |

> 验证码开关由后端配置项 `sys.account.captchaEnabled` 控制（Redis key）。

---

### 成功响应（HTTP 200）

```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 状态码，200=成功 |
| `msg` | string | 响应消息 |
| `data.token` | string | JWT 访问令牌，后续请求需在 Header 中携带 |

**后续请求 Header 示例：**
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

---

### 异常响应

| 状态码 | msg | 原因 |
|--------|-----|------|
| 500 | `账号或密码错误` | 用户名或密码不正确 |
| 500 | `用户不存在` | 账号不存在 |
| 500 | `用户已停用` | 账号状态为停用 |
| 500 | `用户已封禁` | 账号被封禁 |
| 500 | `验证码错误` | 验证码填写错误 |
| 500 | `验证码已过期` | 验证码超时 |
| 500 | `同一账号不允许同时在线` | 开启了单点登录但账号已在别处登录 |

---

### 请求示例

```bash
# 不需要验证码
curl -X POST 'http://10.10.30.31:38038/login' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'username=admin&password=admin123'

# 需要验证码（先调用验证码接口获取 uuid 和 img）
curl -X POST 'http://10.10.30.31:38038/login' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'username=admin&password=admin123&code=abcd&uuid=xxxx-xxxx'
```

## 二、获取当前用户信息

### 接口信息

| 项目 | 说明 |
|------|------|
| **URL** | `GET /getInfo` |
| **认证** | ✅ 需携带登录返回的 Token |

### 请求 Header

```
Authorization: Bearer <token>
```

---

### 成功响应（HTTP 200）

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "permissions": ["system:user:list", "system:user:add", "system:user:edit"],
    "roles": [
      {
        "role_id": 1,
        "role_name": "超级管理员",
        "role_key": "admin",
        "data_scope": "1"
      }
    ],
    "user": {
      "user_id": 1,
      "user_name": "admin",
      "nick_name": "管理员",
      "email": "admin@example.com",
      "phonenumber": "15888888888",
      "sex": "0",
      "avatar": "/profile/avatar/2026/06/08/avatar_20260608083803A048.png",
      "status": "0",
      "dept": {
        "dept_id": 1,
        "dept_name": "深海技术部",
        "leader": "管理员"
      },
      "role_ids": "1",
      "post_ids": "1"
    }
  }
}
```

**响应字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.permissions` | string[] | 权限标识列表，如 `["system:user:list"]` |
| `data.roles[].role_id` | int | 角色ID |
| `data.roles[].role_name` | string | 角色名称 |
| `data.roles[].role_key` | string | 角色关键字，admin=超级管理员 |
| `data.roles[].data_scope` | string | 数据权限范围，1=全部数据权限 |
| `data.user.user_id` | int | 用户ID |
| `data.user.user_name` | string | 用户账号 |
| `data.user.nick_name` | string | 用户昵称/姓名 |
| `data.user.email` | string | 邮箱 |
| `data.user.phonenumber` | string | 手机号 |
| `data.user.sex` | string | 性别，0=未知 1=男 2=女 |
| `data.user.avatar` | string | 头像相对路径，完整访问地址 = 后端地址 + 此路径 |
| `data.user.status` | string | 状态，0=正常 1=停用 |
| `data.user.dept` | object | 所属部门 |
| `data.user.dept.dept_name` | string | 部门名称 |
| `data.user.role_ids` | string | 角色ID列表（逗号分隔） |
| `data.user.post_ids` | string | 岗位ID列表（逗号分隔） |

---

### 异常响应

| HTTP Status | code | msg | 原因 |
|-------------|------|-----|------|
| 401 | 401 | `Token已失效，请重新登录` | 未携带Token或Token格式错误 |
| 401 | 401 | `Token已过期，请重新登录` | Token已过期 |
| 500 | 500 | `获取用户信息异常` | 用户信息查询异常 |

**未授权响应示例：**
```json
{
  "code": 401,
  "msg": "Token已失效，请重新登录",
  "data": null
}
```

---

### 请求示例

```bash
curl -X GET 'http://10.10.30.31:38038/getInfo' \
  -H 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

## 五、通用响应格式

所有接口统一封装在 `ResponseUtil` 中：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | 业务状态码，200=成功，500=失败，401=未授权 |
| `msg` | string | 中文描述信息 |
| `data` | object/null | 业务数据体，失败时为 null |

**code 与 HTTP Status 对照：**

| HTTP Status | code | 含义 |
|-------------|------|------|
| 200 | 200 | 请求成功 |
| 401 | 401 | 未认证 / Token 无效 |
| 500 | 500 | 服务器异常 / 业务异常 |

### 权限数据结构说明

`permissions` 数组中的每个元素格式为 `{module}:{resource}:{action}`，含义如下：

| 段 | 含义 | 示例 |
|----|------|------|
| 第一段 | 所属模块 | `system`=系统管理，`monitor`=监控管理，`warning`=预警 |
| 第二段 | 资源对象 | `user`=用户，`dept`=部门，`post`=岗位等 |
| 第三段 | 操作类型 | `list`=列表 `query`=查询 `add`=新增 `edit`=编辑 `remove`=删除 `export`=导出 |

前端通过 `permissions` 数组判断当前用户是否有权执行某个操作。

### 权限判断代码示例

```python
from utils.permission_util import PermissionManager

# 检查是否有某权限
if PermissionManager.check_perms('system:user:edit'):
    # 显示编辑按钮
    pass

# 当前用户所有权限
permissions = CacheManager.get('permissions')  # list 类型
```
